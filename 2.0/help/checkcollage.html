Validating patterns
Previous  Top  Next


You can validate the pattern for consistency, to avoid the following situations:
·Tiles cover each other;  
·Tiles in the pattern are lying not tightly, with spacings;  
·Neighboring pattern fragments cover each other;  
·Neighboring pattern fragments are lying not tightly, with spacings.  

To perform the check, click the button on the toolbar:


The program will create a plot (2x2 meters) and will pave it with tiles, checking for errors described above. If all parameters are correct, the following window will pop-up:


If any of the points of the neighboring patterns are set incorrectly, you will see the following window:


Visual effects will help you analyze the errors. Voids are colored in blue; areas where tiles cover one another are colored in red. To avoid the draft overloaded with red lines, the program displays the basic areas where tiles cover neighboring tiles. Once you correct this error, it will be fixed all over the plot.

Close the window and correct the coordinates of the points on the pattern. Then perform the check once again.
